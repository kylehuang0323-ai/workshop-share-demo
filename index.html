演示说明

  • 建议 Prompt 保持英文，兼容性更稳定
  • 每题先跑简易版，再跑进阶版
  • 目标是“现场出结果”，不是只看流程

演示 1 · 用户反馈智能分类

中英混合反馈分流任务：给 300 条反馈打上类型、严重程度、主题、语言标签，并输出结构化 CSV。

输入

`feedback.csv`（两列：id / feedback）

输出

`prediction.csv`（id, type, severity, theme, lang）+ 分类分布摘要

考察能力

中英双语多标签分类、模糊语义判断、文件输出。

简易

Read feedback.csv (300 user feedback rows for a product, mixed English and Chinese). Classify each row by type, severity, theme, and lang. Save to prediction.csv with columns id, type, severity, theme, lang, then give a short summary.

进阶

I have 300 mixed-language feedback entries in feedback.csv. Classify each item by type (bug / feature_request / complaint / praise / question), severity (critical / high / medium / low), theme (UI/UX / performance / billing / onboarding / functionality / documentation), and lang (en / zh). Save as prediction.csv and summarize count, type/severity distribution, and hard-to-classify cases.

演示 2 · 周报生成（多 Agent）

同源数据输出三份不同受众周报：工程、管理、全员。重点是角色拆分与并行写作。

输入文件

`mock-ado-workitems.json` / `mock-emails.json` / `mock-meetings.json` / `mock-teams-messages.json`

输出文件

`report-engineers.md` / `report-leadership.md` / `report-team.md`

考察能力

Multi-agent 编排、受众感知写作、一次性流程沉淀为可复用技能。

简易

Using this week's work items, email summaries, meeting notes, and team highlights, generate a concise report.md covering releases, key decisions, risks, and team wins.

进阶

Orchestrate a multi-agent workflow: one Analyzer agent reads the 4 input JSON files and creates a structured weekly summary; three Writer agents produce report-engineers.md, report-leadership.md, and report-team.md in parallel; one Reviewer agent outputs a 5-line executive summary and flags inconsistencies across the 3 reports.

演示 3 · 流程沉淀为可复用 Skill

把“本次跑通”的多 Agent 周报流程，沉淀成下次可一键复用的技能定义文件。

进阶

Now convert the weekly reporting workflow into a reusable skill named weekly-report-generator. Generate SKILL.md with trigger scenarios, required inputs, numbered multi-agent steps, output file naming conventions, and one runnable example. Save it to .skills/weekly-report-generator/SKILL.md.

进阶赛道（3 选 1）

三条赛道分别锻炼：证据化问答、长文结构化、演示表达压缩。可按团队角色自由选题。

赛道 A：视频问答

输入视频 + 问题，输出带时间戳证据的严格 JSON（answer / evidence / confidence / reasoning）。

能力：证据约束与防幻觉

赛道 B：视频转技术博客

输出自包含 blog.html，含 TL;DR、3+ 章节、2+ 关键帧图片和时间戳引用。

能力：长文组织与视觉压缩

赛道 C：视频转汇报 PPT

输出 5-7 页 summary.pptx，要求统一配色、非纯文字版式、关键帧素材与演讲备注。

能力：信息提炼与表达
