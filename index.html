全局说明：这场分享会带大家完成什么

先完成 3 个核心 Demo（分类、周报、多 Agent 沉淀），再选 1 条进阶赛道（视频问答 / 博客 / 演示文稿）。每一步都给“可直接复制”的 Prompt 模板。

阶段 1：快速建立信心

通过一个中英混合的真实分流任务，让参与者 10-15 分钟内拿到第一份高质量输出。

关键词：结构化输出

阶段 2：扩展到团队协作

同一份输入数据，输出 3 份面向不同受众的内容，体现 Agent 编排的业务价值。

关键词：多受众写作

阶段 3：沉淀可复用能力

把一次性 Prompt 进化为可重复调用的 Skill，解决“会演示但难复现”的痛点。

关键词：资产化

阶段 4：按角色选择进阶赛道

根据团队职责进入视频问答、技术博客或 PPT 赛道，形成“当天能产出”的闭环。

关键词：角色适配

核心 Demo（文字版）

以下内容按原始顺序重组为文字版任务卡，已替换成 RE&F 场景用语。

Demo 1

用户反馈智能分类（中英混合）

目标：把 300 条反馈打上 type / severity / theme / lang 四类标签，并导出 CSV。

输入：`feedback.csv`（中英混合、长度不一）。输出：`prediction.csv` + 分布摘要。

Read feedback.csv and classify each row by type, severity, theme and lang. Save to prediction.csv and provide a short summary with distribution and hard-to-classify cases.

Demo 2

多受众周报生成（多 Agent）

目标：同源数据生成工程版、管理版、团队版 3 份周报。

输入：工单、邮件、会议、Teams 四类 JSON。输出：3 份 Markdown 报告。

Orchestrate analyzer + 3 writer agents to produce report-engineers.md, report-leadership.md and report-team.md from the same weekly source files.

Demo 3

把流程沉淀成可复用 Skill

目标：将多 Agent 周报流程沉淀为 `weekly-report-generator`，下周可一键复跑。

输入：4 个数据文件或一个目录。输出：`.skills/weekly-report-generator/SKILL.md`。

Generate SKILL.md for a reusable weekly-report-generator with trigger rules, required inputs, workflow steps and expected outputs.

进阶赛道（3 选 1）

按角色挑选赛道：问答分析、知识沉淀、汇报表达。避免“学完但不会用”的断层。

赛道 A：视频问答

从视频中抽取证据，按严格 JSON 输出答案、关键点、时间戳与置信度，控制幻觉。

适合：分析型角色

赛道 B：视频转技术博客

把视频内容压缩为结构化 HTML 博客，带 TL;DR、章节、关键帧与时间戳引用。

适合：知识运营/文档负责人

赛道 C：视频转汇报 PPT

输出 5-7 页演示稿，包含对比/流程类版式、关键帧素材和演讲备注。

适合：分享者/项目经理

这套节奏为什么适合 RE&F

它覆盖了 RE&F 常见的三类工作：信息分流、跨角色沟通、结构化汇报，且每一类都有可复跑模板。

可见收益
每位参与者都能在现场得到一份可用产出
会后复现
带走 Prompt 与模板，1 周内可在真实任务复做
组织层价值
把“个人经验”转成“团队可共享资产”

90 分钟带练节奏

1. 10 分钟：统一目标

先定义“本场要拿到什么产出”，再开始操作，避免偏离目标。

2. 55 分钟：3 个核心 Demo

你演示一轮，参与者复做一轮，确保不是“只看不练”。

3. 20 分钟：进阶赛道冲刺

按角色分组，快速完成 A/B/C 赛道中的一个落地产物。

4. 5 分钟：复盘承诺

每人承诺一个“下周必用”任务，把学习转化为行动。
